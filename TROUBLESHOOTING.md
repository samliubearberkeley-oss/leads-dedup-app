# 故障排查指南

## 如果点击按钮没反应

### 1. 检查浏览器控制台
- 按 `F12` 或 `Cmd+Option+I` 打开开发者工具
- 查看 Console 标签是否有红色错误信息
- 如果有错误，请截图发给我

### 2. 检查网络请求
- 在开发者工具中打开 Network 标签
- 刷新页面
- 查看是否有失败的请求（红色）
- 特别是检查 `g7str5qu.us-east.insforge.app` 的请求

### 3. 清除缓存
- 按 `Cmd+Shift+R` (Mac) 或 `Ctrl+Shift+R` (Windows) 强制刷新
- 或者在浏览器设置中清除缓存

### 4. 检查文件上传
- 确保CSV文件格式正确
- 文件必须包含 `URL` 列
- 文件大小不要太大（建议 < 10MB）

### 5. 常见问题

#### 问题：点击标签页没反应
**解决方案：**
- 检查浏览器控制台是否有错误
- 尝试刷新页面（Cmd+R）
- 检查网络连接

#### 问题：上传文件没反应
**解决方案：**
- 确保文件是 `.csv` 格式
- 检查文件是否包含表头
- 查看浏览器控制台的错误信息

#### 问题：数据不显示
**解决方案：**
- 点击右上角的刷新按钮
- 检查网络请求是否成功
- 查看控制台是否有错误

### 6. 测试步骤

1. **测试标签切换**
   - 点击"上传/粘贴" → 应该显示上传界面
   - 点击"新Leads" → 应该显示新leads列表
   - 点击"已发送" → 应该显示已发送列表

2. **测试文件上传**
   - 准备一个CSV文件
   - 拖拽到上传区域
   - 应该显示处理结果

3. **测试搜索**
   - 在"已发送"页面
   - 输入关键词搜索
   - 应该实时过滤结果

### 7. 如果还是不行

请提供以下信息：
1. 浏览器类型和版本
2. 控制台的错误信息（截图）
3. Network标签中的失败请求
4. 具体哪个按钮没反应

## 快速修复命令

```bash
# 重启开发服务器
cd leads-dedup-app
pkill -f vite
npm run dev

# 清除node_modules重新安装（如果问题持续）
rm -rf node_modules package-lock.json
npm install
npm run dev
```
